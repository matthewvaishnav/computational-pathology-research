## 📋 Detailed Deployment Guide

### Step 1: Prepare Environment

#### Build Docker Image
```bash
# Create Dockerfile (auto-generated by deploy script)
python scripts/deploy_k8s.py --build-image

# Or manually:
docker build -t medical-ai:latest .
```

#### Configure Secrets
```bash
# Edit secrets (IMPORTANT: Change in production!)
vim k8s/secrets.yaml

# Update these base64 encoded values:
# - POSTGRES_PASSWORD
# - DATABASE_URL  
# - SECRET_KEY
# - JWT_SECRET_KEY

# To encode secrets:
echo -n "your-secret" | base64
```

### Step 2: Deploy Infrastructure

#### Deploy Core Components
```bash
# 1. Namespace and RBAC
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/rbac.yaml

# 2. Configuration
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/secrets.yaml

# 3. Database
kubectl apply -f k8s/postgres.yaml

# Wait for database
kubectl wait --for=condition=available deployment/postgres -n medical-ai --timeout=300s
```

#### Deploy Application
```bash
# 4. API Application
kubectl apply -f k8s/api-deployment.yaml

# 5. Services and Ingress
kubectl apply -f k8s/services.yaml

# Wait for API
kubectl wait --for=condition=available deployment/medical-ai-api -n medical-ai --timeout=600s
```

### Step 3: Deploy Monitoring

#### Monitoring Stack
```bash
# Deploy monitoring components
kubectl apply -f k8s/monitoring/prometheus.yaml
kubectl apply -f k8s/monitoring/grafana.yaml
kubectl apply -f k8s/monitoring/alertmanager.yaml

# Wait for monitoring
kubectl wait --for=condition=available deployment/prometheus -n medical-ai --timeout=300s
kubectl wait --for=condition=available deployment/grafana -n medical-ai --timeout=300s
```

### Step 4: Enable Auto-scaling

#### Horizontal and Vertical Scaling
```bash
# Deploy auto-scaling
kubectl apply -f k8s/hpa.yaml
kubectl apply -f k8s/vpa.yaml  # If VPA is available

# Verify HPA
kubectl get hpa -n medical-ai
```

### Step 5: Security Hardening

#### Network Policies and Security
```bash
# Deploy security policies
kubectl apply -f k8s/security/pod-security-policy.yaml

# Verify network policies
kubectl get networkpolicy -n medical-ai
```

## 🔧 Configuration Options

### Environment Variables

#### API Configuration
```yaml
# In k8s/configmap.yaml
API_WORKERS: "4"          # Number of API workers
LOG_LEVEL: "INFO"         # Logging level
MAX_FILE_SIZE_MB: "100"   # Max upload size
```

#### Resource Limits
```yaml
# In k8s/api-deployment.yaml
resources:
  requests:
    cpu: 1                # Minimum CPU
    memory: 2Gi           # Minimum memory
  limits:
    cpu: 4                # Maximum CPU
    memory: 8Gi           # Maximum memory
```

#### Auto-scaling Thresholds
```yaml
# In k8s/hpa.yaml
minReplicas: 3            # Minimum pods
maxReplicas: 20           # Maximum pods
targetCPUUtilization: 70  # CPU threshold
targetMemoryUtilization: 80  # Memory threshold
```

### Storage Configuration

#### Database Storage
```yaml
# In k8s/postgres.yaml
storage: 20Gi             # Database storage size
storageClassName: fast-ssd  # Storage class
```

#### Model Storage
```yaml
# In k8s/api-deployment.yaml
storage: 10Gi             # Model storage size
accessModes: ReadOnlyMany # Shared across pods
```

## 🚀 Scaling and Performance

### Horizontal Pod Autoscaler (HPA)

#### API Scaling
- **Min Replicas**: 3 (high availability)
- **Max Replicas**: 20 (handle traffic spikes)
- **CPU Target**: 70% utilization
- **Memory Target**: 80% utilization
- **Custom Metrics**: HTTP requests per second

#### Database Scaling
- **Min Replicas**: 1 (primary)
- **Max Replicas**: 3 (read replicas)
- **CPU Target**: 80% utilization

### Vertical Pod Autoscaler (VPA)

#### Resource Optimization
- **Automatic resource adjustment** based on usage patterns
- **Right-sizing** to optimize costs
- **Performance optimization** with appropriate limits


## 🔧 Troubleshooting

### Common Issues

#### 1. Pods Not Starting
```bash
# Check pod status
kubectl get pods -n medical-ai

# Check pod logs
kubectl logs -n medical-ai deployment/medical-ai-api

# Check events
kubectl get events -n medical-ai --sort-by='.lastTimestamp'
```

#### 2. Database Connection Issues
```bash
# Check database pod
kubectl logs -n medical-ai deployment/postgres

# Test database connectivity
kubectl exec -n medical-ai deployment/medical-ai-api -- pg_isready -h postgres-service -p 5432
```

#### 3. Model Loading Issues
```bash
# Check model storage
kubectl exec -n medical-ai deployment/medical-ai-api -- ls -la /app/models/

# Check model loading logs
kubectl logs -n medical-ai deployment/medical-ai-api | grep -i model
```

#### 4. Auto-scaling Not Working
```bash
# Check HPA status
kubectl get hpa -n medical-ai

# Check metrics server
kubectl top pods -n medical-ai

# Check HPA events
kubectl describe hpa medical-ai-api-hpa -n medical-ai
```

### Debug Commands

#### Resource Usage
```bash
# Pod resource usage
kubectl top pods -n medical-ai

# Node resource usage
kubectl top nodes

# Detailed pod info
kubectl describe pod -n medical-ai -l app=medical-ai-api
```

#### Network Connectivity
```bash
# Test service connectivity
kubectl exec -n medical-ai deployment/medical-ai-api -- nslookup postgres-service

# Test external connectivity
kubectl exec -n medical-ai deployment/medical-ai-api -- curl -I https://google.com
```
